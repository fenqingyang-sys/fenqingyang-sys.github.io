inline function

Posted on 2019-10-31 | In c++ | | Visitors:

当程序执行函数调用指令时，CPU将存储该函数调用之后的指令的内存地址，将函数的参数复制到堆栈上，最后将控制权转移到指定的函数。然后，CPU执行功能代码，将功能返回值存储在预定义的存储位置/寄存器中，并将控制权返回给调用函数。如果函数的执行时间少于从调用者函数到被调用函数（被调用者）的切换时间，则这可能会成为开销。对于大型函数和/或执行复杂任务的函数，与函数运行所花费的时间相比，函数调用的开销通常很小。但是，对于常用的小型函数而言，进行函数调用所需的时间通常比实际执行函数代码所需的时间长得多。对于小型功能，会发生这种开销，因为小型功能的执行时间小于切换时间。c++的内敛函数解决函数调用时间大于函数执行时间的问题，从而提高程序的运行效率。

内联函数的特点：

1内敛函数是在编译的时候进行。

内敛函数的缺点：

2内联函数中添加的变量会消耗更多的寄存器，在内联函数中，如果要使用寄存器的变量编号增加了，这可能会增加寄存器变量资源利用率。 这意味着当在函数调用点替换内联函数主体时，该函数使用的变量总数也会被插入。 因此，将用于变量的寄存器数量也将增加。 因此，如果函数内联后的变量数急剧增加，则肯定会导致寄存器利用率增加。

3如果使用太多的内联函数，则由于重复执行相同的代码，因此二进制可执行文件的大小将很大。

4过多的内联也会降低指令高速缓存命中率，从而降低了从高速缓存到主存储器的指令获取速度。

5如果有人更改了内联函数中的代码，则内联函数可能会增加编译时间开销，因此必须重新编译所有调用位置，因为编译器将需要再次替换所有代码以反映更改，否则它将继续使用旧功能 。

6内联功能对于许多嵌入式系统可能没有用。 因为在嵌入式系统中，代码大小比速度更重要。

7内联函数可能导致系统性能下降，因为内联可能会增加二进制可执行文件的大小。 内存溢出会导致计算机性能下降

最后内联函数是C ++的宝贵功能。 适当地使用内联函数可以提高性能，但是如果任意使用内联函数，则它们无法提供更好的结果。 换句话说，不要指望程序会有更好的性能。 不要使每个函数都内联。 最好使内联函数尽可能小。

一

师夷长技以制夷，king决定学习来自东方（准确来说是日本的西方）的哲学。他去东京大学中文系旁听，总坐在第一排，那些占座的前一天晚上放在桌子的书，他见了只管摞起来，更有甚者，有些人放了一支笔就代表占座，king见四下无人，寻不着失主，便把笔放到讲台桌上。次日，那些人惊诧的发现笔不见了，就问king：“你瞧着我的笔了吗？“，king说，讲台桌上，难怪找不到失主，原来是你放在这的。又过了几天，那人重蹈覆辙。笔还是不见了，见king坐到她占的位置，便问：”你这人怎么不识趣呢？明明是我占的位……“说着便呜咽起来。king头一甩，只当没看见。其他一些爱打抱不平的人便凑了过来，其中有个人穿皮夹克，个头不高，他眉头紧锁，用手指指向king，说到：”众目睽睽，你怎么欺负人呢？“说完，不问青红皂白，抡起衣袖，重拳出击，king支出一根手指，便将那蛮力轻松移转。那人有点意外，顺势将挥出去的手迂回到头上，摸摸脑袋，问那个占座的人：”你们，有什么矛盾？“占座的人理直气壮地说：”占了座，不该就是我的座位吗？“

king冷冷地说：“从来如此，就对吗？适可而止吧！”，霎时，室温似降了几十度，冻得在场的人瑟瑟发抖。皮夹克男觉得他的皮夹克结了冰，手脚也僵硬的不能动弹。在场的人沉默了许久，身体才缓和过来。上课铃响了，众人不欢而散。

这个皮夹克男名叫木，喜欢西洋文化，热衷健身，总在幻想凭借拳头解决事情，但每次总被教训一顿。就像上周，他看见一群穿着皮夹克的飙车族在街头向一个手无寸铁的小朋友要保护费，看到这，他心血来潮，便飞奔回家里，套上他父亲或者他爷爷穿过的皮夹克，没准别人会看皮夹克的成色来给保护费呢，想到这，他准备横征暴敛，刚走出街区，便发现一个从拐角出来的小孩。他两腿就好像有使不完的劲儿，直奔小孩而去。木走到他面前，支支吾吾的蹦出几个字：“收保护费。”，摊开他的右手，（这还是他第一次打劫）当他看到拐角的黑影时，已经迟了。眼睛挨了一拳，脑子直冒金星，还没回过神来，又挨了一拳，这拳似乎比上一拳更有力，因为刚刚这一拳把他打得不省人事。他一个趔趄，往后退了几步，之后如同脚底生风，溜之大吉，回头只瞥见小孩疑惑的脸和一只在他头上方粗壮的手。

似乎有一只脚在铃响完的瞬间迈入教室。衣着绿白相间花格子衬衫，个头很挺拔，气宇轩昂，脸色是中年男子特有的黑色。上课总是那么无聊，这个老师讲的在座的所有人都昏昏欲睡，木趴在桌子上，不知道进入到睡眠的第几个周期了。只有king正襟危坐，手上不停的记着笔记，也许老师说了多少话他便记了多少了字，同声传录的技术还是ov教给他的。king记性是真的糟糕，因为他实在记不起放了一支笔在桌子上就代表占座，前天刚讲过的话在脑海中早就没了影子，如果他不一字一句地做笔记，没准哪天就忘了字怎么写。

木好像有未卜先知的能力，总在下课铃响起前几分钟惊醒，就好像在体内安装了定时闹钟。他挣扎着从位子上爬起，看到其他人陆陆续续离开教室，只有king呆若木鸡的坐在那。木心想这个人非等闲之辈，决定不去惹恼他，悄溜溜的离开教室。一改教室里的压抑和沉闷，外面阳光明媚，路旁的樱花树随风起舞，被吹起的樱花随风飘散，它们旋至高空后七零八碎。

木可没注意到这些，他只觉得这个天气穿皮夹克刚刚好，既不冷也不热。剩下的时间没课，他便找个地解闷去了

king在座位上回味着来自东方的智慧，但他不懂这些人为什么这么喜欢日，或许只有干那事才能领悟人生哲理？他有点蠢蠢欲动。king合上笔记本，夹在腰际。他刚走出教室门，眼光就停留在一个熟悉的身影，他定睛一看，确认无疑，她就是ov，但不同的是，他注意到她身后跟着两个虎背熊腰的壮汉，尽管西装革履，他还能一眼看出他们的痞气。无论是从生理还是心理，他再也没有勇气去接近ov了。如果他能比那些保镖更强壮，比doge更强大，他必然将本属于他的东西夺回来。这便是他目前的人生目标了。人生下来便有了目标，假如今天没有，那明天就多了一份有的可能。假如现在没有，将来也会有。如果一直没有呢？那他便是一只可怜的糊涂虫了，他的人生只能得过且过了。

“真的？赶紧去正行鸡店买个一车！“

木听到竟信以为真，原来他之前挨揍的原因就在这里，他没有吃足够的鸡排！他将杯中的咖啡一饮而尽，苦涩与泪水都在肚子里流。他打开手机，导航到最近的鸡店。点了一份鸡排,又加了几块钱买了杯灌水的可乐。等油炸鸡排机‘叮’的响了一声。木知道，他来了。虽迟但到。服务员和蔼的递给了木鸡排。木颤抖着接过香喷喷的鸡排，这，难道就是鸡排？他感到异常紧张，拿起竹签慢慢放入嘴中，这块鸡排烫的像嘴里点了一把火，他胡乱吃了几口就咽下去了。眼前好像飘过一幅画面，一个人乘着一叶扁舟，碧绿的河水在船下流淌，它向远处延伸，最后在一片茂密的桦树林后失去踪影。那人撑着舟，也拐入树林之后。画面戛然而止，木感觉愣了很久，咽了咽口水，已没鸡排味，画面结束之后才回过神来。这感觉就像在看全息投影。他接着往嘴里塞下一块鸡排，神奇的是，画面也接着下去。那人蒙着眼睛，神情凝重，腰间系一把刀，如果没看错，那是一把杀猪刀，都说杀鸡焉用宰牛刀，可没说不能用杀猪刀。我想那个人也是这么想的。他撑着船，远远的好像看见一个草棚。没猜错的话，那就是鸡场了。古传，鸡霓泰昧之地。(也许有这么个传言)上古歌神毝（cai）勖（xu）貇因触犯天条被发配到鸡霓泰昧看守鸡场，等到那里的鸡吃完了米山他才能重返天庭。米山绵延千里，这下，就算有再多鸡也吃不完了。之后由于板块运动以及分子间作用力化成了一座真的山，后人通过牺牲成百上千人生命的努力才测量出他的实际高度，8848米(日本国家地质局认可的精确数据)，由于食物充足以及没有天敌迫害，鸡大量的繁殖，大量的鸡屎也堆积起来，日久天长，鸡屎块结，也成了一座山，但后来就不知道哪去了。这我就不再细谈。接着说那些鸡，由于鸡的泛滥，人们开始大量的捕杀鸡，其中一些人变成了前文所诉的鸡排鬼。而这个乘舟的人是鸡排鬼中最杀戮成性的家伙，死在它刀下的不计其数，在临死前那些鸡会嘶喊:”＊＊”，这次他竟想直捣黄龙，把鸡霓泰昧之地一锅端。毝勖貇本是上古歌神，再加上每日闻鸡起舞，舞艺相比也日益臻进，这是个棘手的家伙。那人想着(也许是木想着)。

画面又到此为止，木又像一头蠢驴一样愣了半晌，直到别人拍了拍他才come to life。那人找他问路，他皱着眉头，没好气的说:”没看到我在想事吗？”。那人摇了摇头离去了。木在袋中摸索鸡排，哑然一惊，发现早已空无一物。木感觉已经快接近事物的真相了，赶忙跑到鸡店续上一份鸡排。他继续吃，画面却不再浮现。他有点沮丧。可乐里的冰已化为水，从杯子里满了一部分出去。现在水的比例又高了。为了去除满嘴的油腻，勉为其难咽了一小口，漱漱口，啖了一口，感觉比之前更难受。把杯子随地一扔。走了。

夜幕降临，king在归家途中闻到百米之外鸡店飘香，这唤醒他体内沉睡的细胞。他咂咂嘴，吞咽贪婪的口水。匆匆跑开。

在家里换上作案装备，把黑布蒙到眼睛上。在这里需要补充的是:由于生态选择，鸡排鬼已经进化到能够凭借气味来判断物体的准确位置。他之前路过的黑店首先被他关掉电源，这下就能让那些凭借视觉感知事物的店员慌了神，四肢瘫软，那些店员就跟锅里的鸡排一样好收拾了，轻而易举的被king击倒，这时店员就不敢乱动了，只能在黑灯瞎火中听到鸡排被咀嚼的清脆声以及冷冻鸡排互相碰撞渐行渐远的声音。

鸡排鬼的进化说明了。有的人不再相信眼睛，只怕眼中充满欺骗。

银河补习班--关于教育的看法

Posted on 2019-09-25 | In 观影 | | Visitors:

国庆期间在享受小假期的欢愉之外看了一部电影——《银河补习班》，总体上感觉影片质量还行，观看之后不仅再次想起关于家庭教育的一些问题以及自己的一些看法。记得自己第一次接触关于孩子的教育的书籍便是周国平的《宝贝宝贝》和《妞妞》，依稀记得自己第一次看完时的感受，感觉父亲在一个孩子的成长中扮演了多么重要的角色。孩子的童年如果缺乏父母的呵护与陪伴，这样的童年是不完整的童年。于是不仅思考当下现在这个快速发展的时代，无数的家长为了能够给自己的孩子一个更好的经济条件而不能陪伴孩子成长，这是多么令人痛心！再回想电影《何以为家》一个控告自己父母生了自己的孩子，在惊讶和伤痛之余，我们更应该思考这背后的原因，我们都有为人父母的那一天。当那一天来临之际，身为一个父母的你有没有身为父母的资格？希望每个父母在成为父母的那一刻可以将自己的孩子看成是一个有独立灵魂的人，我们不应该把我们的意志强加到孩子的身上，我们不应该将他仅仅视为一个孩子，他们是独一无二的，应该被看作一个独立的个体。这样当和孩子相处的时候你们是近似对等的更多是一种朋友和引路者的身份，没有居高临下的那种绝对权威，也不会什么都以自己的标准来要求孩子而忽略他们的感受。我希望自己在孩子成长的路上不会缺席，但是只是以一个引路人的身份在背后默默陪他走完他人生的各个阶段。

很喜欢影片中的一些话，这个世界上有这么多美好的有意思的工作，挑一个你最喜欢的，一直想。兴趣真的是孩子学习的原动力，是不断学习终身学习的助推剂。我觉得在当下这个快速发展的世界中，有这么多的机会和工作。你做自己喜欢做的事情，做到极致，我觉得还是可以养活自己的。所以以后当你成为父母我希望你可以给孩子自己选择的机会，人的一生说长不长说短不短，他们应该活出自己的样子而不是别人家的孩子的样子，也不是我们作为父母强加给孩子让孩子活成我们曾经想活成的样子。要培养孩子独立思考和选择的能力，让他们学会为自己的选择承担相应的责任。父母不可能陪伴你走完一生，不仅仅是父母，谁都不能陪伴你从开头到结尾走完一生。每个人都只是陪你走完你人生路上的某一个阶段。所以他们要学会自己做出自己的选择，学会思考自己的事情。每个孩子的特质都不一样，对于孩子表现出的特质我们应该给呵护，比如孩子天马行空的想象力，与人交际的能力等等，都不应该被我们有意或无意的扼杀。最后希望每一对父母都能进到自己的责任，每一个孩子都可以健康快乐的活出他们本来该有的样子。

为什么数组下标一般从0开始？

Posted on 2019-09-18 | In 数据结构与算法之美 | | Visitors:

我们通常会发现大部分的语言数组下标都是从零开始的，我们从数组存储的内存模型上来看，“下标”最确切的定义应该是“偏移”（offset），假如我们用a来表示数组的首地址，a[0]就是偏移为0的位置也就是首地址，a[k]是偏移量k个数据类型所对应的字节的位置（type_size）。a[k]内存位置的计算公式为：

‘a[k]_address=base_adress+k*type_size’

如果数组从1开始计数，计算内存位置的公式为：

‘a[k]_address=base_adress+（k-1）*type_size’

从公式可以看出第二个公式每次计算内存地址时都要多做一次减法，对CPU来说就是多了一条减法指令。而数组作为最基础的数据结构，通过下标访问数组是经常使用的编程操作，所以性能和效率的优化要尽可能做到极致。当然很大原因还是与c语言最初的设计者使用0作为数组下标的起点有关，后续语言基本沿袭这个习惯。但是也有些语言的数组下标不是从0开始的比如MATLAB语言数组下标从1开始，python语言数组下标允许负值。
