git命令的使用

Posted on 2019-09-07 | In 命令、语法 | | Visitors:

git init:在当前目录新建一个Git代码库

git clone [url]：将项目和完整代码克隆到当前所在工作目录

git add [文件名]：将对应的文件上传到暂存区

git commit -m [提交文件的描述信息]：将暂存区的文件上传到仓库

git push origin dev:该命令是将本地dev分支推送到远程仓库对应的远程仓库。
git remote：查看远程仓库的信息。

注：当你从远程仓库克隆时，实际上Git自动把本地的master分支和远程的master分支对应起来了，并且，远程仓库的默认名称是origin。

git status：查看变更的文件详细信息一般用来辅助查看git add和git commit -m[message]所对应的上传信息是否成功。

注：一般可以将对应的github仓库克隆到本地，然后将要上传到github的文件添加到该克隆文件夹下，该文件夹下还会有一个隐藏的.git文件，然后使用git add . 命令可以将本地文件夹下的所有文件上传到该github远程仓库。其中.类似与linux命令下.含义相同代表着当前目录，因此 git add .的含义是将当前目录下的所有文件添加到暂存区。

git diff [文件名]:如果你修改了一个文件几天之后你忘记了修改内容，那么可以使用git diff 命令来查看上一次修改情况。

git log:显示从最近到最远的提交日志，如果日志较多可以采用git log –pretty=oneline,出现的一串串数字是git的commit id

git reset –hard HEAD^:其中HEAD表示当前版本，整个命令表示把当前版本回退到上一个版本，HEAD^表示上一个版本，HEAD^^表示上两个版本，如果回退到20个版本可以使用HEAD~20回退到上一个版本之后，这时候使用git long你会发现原来最新的版本没有了，这个时候如果你后悔了，想回到原来的最新版本，此时如果你的命令窗口没有关闭你可以通过往前翻命令找到原来最新版本的commit id使用git trset –hard [commit ID]回到原来最新的版本。如果你关闭了命令窗口，可以通过使用git reflog命令来查看你原先的命令记录历史从而找到 commit ID回退到最新版本

git checkout –[文件名]：把文件在工作区的修改全部撤销，分成两种情况：一种是文件从修改之后还没有还没有被放到暂存区，这时候撤销修改就是回退到原先未修改的状态。一种是文件已经添加到暂存区，又作了修改，这个时候撤销修改就是回到添加到暂存区时的那个最初状态。git reset HEAD[文件名]可以把暂存区的修改撤销掉重新放回工作区。

git rm [文件名]：删除文件这个时候如果使用git status会告诉你文件被删除了，如果确认删除则使用git commit来提交即可。如果误删除了，可以使用git checkout –[文件名]既可以恢复原先删除的文件。该操作实际上是使用远程仓库里面的文件替换工作区的文件。

git branch:查看分支状态。

git branch [名称]：创建一个分支

git checkout -b [名称]或者是git switch -c[名称]：切换分支

注：git switch不适合旧的git版本。

git checkout -b[名称]或者git switch -c[名称]:创建+切换分支的命令集合。

git merge [名称]：合并某分支到当前分支。

git branch -d[名称]：删除某分支。

注:合并分支适合于我们准备开发一个项目代码没写完提交的话会影响别人的工作，不提交害怕代码丢失，这个时候就可以使用分支管理，创建一个属于自己的分支，你在自己的分支上干活，别人看不到，想提交提交，而且别人还可以在你原先的分支上继续工作，当你完成项目的时候，一次性合并到原先的分支即可。分支合并有时候会出现冲突，这个时候我们首先要先解决冲突在提交。

git stash:将当前工作现场储藏其来，等待以后恢复现场继续工作。

git stash list：查看储藏起来的工作现场。

git stash apply:恢复储藏的工作现场，但是这个时候你在git stash会发现原先储藏的那个工作现场并没有被删除，如果使用git stash drop:就可以在恢复工作现场的同时删除stash的内容。

注该命令一般适合场景为你正在做项目，老板让你在两个小时内修改一个文件的bug，你当前项目还要一段时间完成这个时候你就可以先保存当前的工作场景，在完成老板的任务之后恢复工作场景继续做自己的项目。

git cherry-pick[文件名或者commit ID]：复制特定的一个提交到当前分支。

注：如果你刚才在别的分支修改了代码现在要求重新在另一个分支修改一遍，笨拙的做法是再重新像刚才那样执行一次修改，简便的方法是使用git cherry-pick[文件名或者commit ID]就可以将在别的分支上修改好的bug文件复制到新的分支。一般使用commit ID。

Hello Programing

Posted on 2019-09-06 | In 思想 | | Visitors:

前段时间无意中刷知乎刷到了一篇文章名字是学习编程的正确姿态，看完之后内心还是很有感触的。一直觉得这个世界上思想之间的碰撞才是最美妙的，在看完这篇文章之后，觉得收获了一些东西。

第一个便是对编程的学习方式，个人觉得现在国内的教学和答主说的很像，给你一本厚厚的教材，老师念着PPT如果你认真的听得话，你可能会听懂，但是你会发现你不会写，如果你不认真听的话，那么你是既听不懂也不会写，于是你去书店满怀希望的去买了好几本厚厚的C语言，真实的情况是那些经典的教材在网上都有PDF格式，而实际上你买的教材在心血来潮的激情过后，他们就在默默的吃灰。然后慢慢地你就会怀疑自己是不是不适合学习编程，学来学去，学着前面的忘着后面的，水平一直得不到增长。如果三年都是这样，那么必然凉凉。

其实佛和魔只在一念之间。学习编程你应该先接触这个环境，你应该自顶向下的去学习编程。你要有自学的能力，要学会使用搜索引擎去解决自己遇到的问题，可以说基本上你遇到的90%
以上的问题都可以在搜索引擎上去解决。你还要学会使用优秀的工具，有时候会发现一个现象当Google可以精准的定位你的问题的时候，很多人还在百度一堆的广告和乱七八糟的结果中找不到问题的答案，当VScode越来越优秀的时候你还在机房里面用着vc6看着密密麻麻的报错，有人说工具不重要重要的是能力，其实工具同样重要，优秀的工具可以让你有一个舒畅的工作环境，可以让你高效的开发。

学习编程首先要打开你的视野，而打开你视野强有力的工具之一是Google。在Google上面你可以通过github上看到这个世界上最牛逼的代码，可以看到很多优秀的项目，你可以在上面用PR提交你遇到的问题，一般你今天提交第二天会收到国外友人的回复，我第一次提交问题在第二天上午看到回复的时候，那一刻很激动。你可以知道你学的知识可以干什么。当你改进或者完成一个项目的时候，那种成就感所带来的的正反馈将会是你不断学习的推进力。另一个除了github的社区便是stackflow,你可以在stackoverflow上面发起你的问题，一般都会有人给你回复。

虽然这些在看过这篇文章之前我都有过类似的体会，作者所提到的工具和社区我也都用过，整篇文章给我最大的触动是一种思想，开发者思想。开发者精神是指, 从学习编程第一天起, 你的目标, 你所做的事, 永远不是以要学会什么为目标, 而是以开发出什么为目标, 深刻意识到你学习的一切, 最终是为了你的开发而服务的。与之对应的便是学生气，从学习编程的第一天起, 就把自己当学生, 上课听讲记笔记为第一位, 记忆知识点为第一位, 买教材读教材为第一位, 追求学会什么, 而不是用它开发出什么。两种思想对应的行为举例，以Java学习为例。学生气的行为模式：记笔记, 追求把这个语言的每个知识点都记得很清楚, 追求一种”内功”的修炼, 在这个过程中, 从来不想着用它去”创造”什么. 在学习的第一天起, 他给自己定下了一个目标:”我这个学期一定要把这个语言的基础打牢, 最后在考试中取得高分, 并且为以后的学习提供更坚实的基础。开发者行为模式：先大体了解一下Java语言在哪些领域比较强势,做一个简单的调研, 得出了Java在Web领域很强势这一个信号, 于是在学习Java的第一天起, 他就给自己定下了一个目标: “我要在半年内通过学习Java, 运用Java做出一个中小型的, 基于Web的企业管理后台”
记住现实世界中的人，处理现实世界中的问题，往往不是一口气完成所有的知识储备，然后一口气顺畅的进行开发进行实践。

经常遇到的问题便是学过的知识忘了怎么办，其实如果你具备开发者精神, 那么你根本不会理会这个问题, 因为, 你开发出的东西, 你做出的产品, 它就是永久存在在这个世界上的, 你的成就感来源于真实的, 具体的, 可持久延续的项目中, 而不是来源于”我学会了什么”.既然我们想获得反馈, 那么一个很现实的建议就是, 在Github上创建你的第一个开源代码仓库, 长期地, 稳定地commit, 当然, 至于这个仓库到底是干啥的, 这个因人而异, 有可能是你自己开发的一个VScode插件, 有可能是你自己写的读书笔记, 在这个不断地commit的过程中, 感受你的代码仓库不断增长不断完善的过程中, 你会获得一个比较持久的反馈和成就感, 一旦有了反馈和成就感, 那么你的学习动力就会一直保持, 也可以从学习中获得快乐.然后你需要一个个人博客，用来记录你的学习历程，每当你学会一个东西，写下来，这个过程就是一个输出的过程，在写的时候你要考虑别人能看懂你的文章，这就需要你自己本身对学习的东西很清楚才能写得清楚。然后当你发现你的个人博客得到很多赞的时候，成就感又会推着你继续学习，你会很快乐。再此简短的回顾一下自己的大一，接触过很多东西，从一开始没怎么接触计算机的纯小白，到使用google与世界接轨，到个人博客的搭建，到vps的搭建和虚拟主机的了解，再到建模和MATLAB的接触，到当下深度学习神经网络的初步体验，到一些常用的国外软件如：twiter,youtube,hbonetflixyoutubemusic，twitch，等的接触和使用。每一个模块回想起来自己都走过很多弯路，都有过痛苦的经历，但是与此同时，我也打开了自己狭隘的视野，开始接触许多新的世界，慢慢开始确认自己想要的是什么，时间如白驹过隙，大一一转眼还是匆匆理我而去，未来还在前方招手，好好学习，好好生活。

markdown基本用法

Posted on 2019-09-06 | In 命令、语法 | | Visitors:

1 标题：#代表一级标题，##则代表两个标题以此类推。

示例：### 这是一个三级标题。

2 引用：在引用文字前面加上>即可。

示例：> 本文引用自coding帮助文档。
补充：区块引用可以嵌套，根据层次加上不同的>。
示例：

这是一级引用。

这是二级引用。

3 列表：列表项目标记通常放在最左边，项目标记后面要接一个字符的空格。
无序列表：使用星号、加号或是减号作为列表标记,标记后加一个空格。
示例：

  • Red
  • Green

有序列表：使用数字接着一个英文句点。
示例：

  1. red
  2. green

补充：如果要在列表项目里面放进引用，需要使用>并且要缩进。

示例：

  • color有以下几种

    blue
    green

代办列表：表示列表是否勾选状态，注意:[]前后都要有空格。
示例：

  • [] 不勾选
  • 勾选

4 代码：把代码包裹在’之间，在这种围栏式代码中，你可以指定一个可选的语言标识符，然后可以为代码着色。
示例：

对于单个代码可以采用两个反引号来标记，示例：

scanf()

5 斜体：使用和_来表示斜体和加粗。
示例：
*斜体
斜体
加粗：
示例：
加粗
加粗

6 自动连接：方括号显示说明，圆括号内显示网址，markdown会自动把它转成链接。
示例：
我的github地址

7 表格;
示例：
First Header | Second Header | Third Header
———— | ————- | ————
Content Cell | Content Cell | Content Cell
Content Cell | Content Cell | Content Cell
也可以让两边的内容对齐，中间内容剧中。
示例：
First Header | Second Header | Third Header
:———– | :———–: | ———–:
Left | Center | Right
Left | Center | Right
8 分割线：
在markdown中，可以使用3个以上-符号做分割线。
示例：
分割线以上内容

分割线以下内容

9 图片：markdown使用类似链接的语法来插入图片，包含两种形似:内联和引用。
内联图片语法为一个！接一个[]，方括号里面是图片的替代文字，再加一个普通圆括号，圆括号里面是图片的网址，最后可以用引号包住并加上选择性的文字。
示例：

![Alt text][https://www.grumpycats.com/]
